# Сборка игры через GitHub Actions

Проект уже настроен для автоматической сборки на GitHub.

## Что собирается

После загрузки проекта GitHub выдаёт два файла:

1. `Football_Manager_2003_Windows.zip` — версия для Windows.
2. `Football_Manager_2003_Android_debug.apk` — тестовый APK для Android.

## Как загрузить проект

1. Создайте на GitHub новый пустой репозиторий.
2. Не добавляйте при создании README, `.gitignore` или лицензию — они уже есть в проекте.
3. Загрузите в корень репозитория **содержимое этой папки**, включая скрытую папку `.github`.
4. Откройте вкладку **Actions**.
5. Выберите workflow **«Сборка игры»**.
6. Дождитесь зелёной галочки.
7. Откройте завершённый запуск и скачайте файлы из раздела **Artifacts**.

## Ручной запуск

Во вкладке **Actions** откройте **«Сборка игры»**, нажмите **Run workflow**, затем ещё раз **Run workflow**.

## Автоматический релиз

Чтобы готовые файлы появились во вкладке **Releases**, создайте тег версии, например:

```bash
git tag v0.9.1
git push origin v0.9.1
```

GitHub автоматически создаст релиз и приложит Windows-архив и APK.

## Важно про APK

Сейчас собирается тестовый debug APK. Он подходит для установки на телефон и проверки игры. Для публикации в Google Play позже потребуется постоянный release-ключ. Его нельзя выкладывать в репозиторий.

## Если сборка красная

Откройте красный запуск, затем шаг с ошибкой и скопируйте его полный текст. По этому логу можно точно исправить проект или workflow.
